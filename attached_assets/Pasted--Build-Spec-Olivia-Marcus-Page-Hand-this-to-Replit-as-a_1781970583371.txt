## Performance: lazy-load the embeds (do NOT load 9 embeds on page load)

Nine Instagram embeds loaded at once will wreck mobile LCP and cause layout shift. Required approach:

1. Server-render a lightweight blockquote for each reel (small HTML, no script):
   ```html
   <blockquote class="instagram-media" data-instgrm-permalink="https://www.instagram.com/reel/DICJ3M6u_zZ/" data-instgrm-version="14" style="min-height:540px;"></blockquote>
   ```
   The inline `min-height` reserves space so there's no layout shift (CLS) when the embed hydrates.

2. Load Instagram's `embed.js` **once, lazily**, only when the reel gallery scrolls into view, then process the embeds. In a Next.js App Router app, put this in a small `'use client'` component:
   ```jsx
   'use client';
   import { useEffect, useRef } from 'react';

   export default function InstagramGalleryLoader() {
     const ref = useRef(null);
     useEffect(() => {
       const el = ref.current;
       if (!el) return;
       const load = () => {
         if (window.instgrm) { window.instgrm.Embeds.process(); return; }
         const s = document.createElement('script');
         s.src = 'https://www.instagram.com/embed.js';
         s.async = true;
         s.onload = () => window.instgrm && window.instgrm.Embeds.process();
         document.body.appendChild(s);
       };
       const io = new IntersectionObserver((entries) => {
         if (entries.some(e => e.isIntersecting)) { load(); io.disconnect(); }
       }, { rootMargin: '300px' });
       io.observe(el);
       return () => io.disconnect();
     }, []);
     return <div ref={ref} aria-hidden="true" />;
   }
   ```
   Render `<InstagramGalleryLoader />` near the gallery. The blockquotes themselves stay in the server-rendered HTML; only the script that turns them into players loads on scroll.

3. If you'd rather go even lighter, use a click-to-load facade: show a thumbnail/play-button card per reel and only inject embed.js + process on click. Either is fine; the IntersectionObserver version above is the simpler default.

## SEO / technical requirements

- Real intro + per-clip captions in the server-rendered HTML (proves on `view-source`). The embeds are enhancement, not the indexable content.
- Self-referencing canonical: `https://survivorhoboken.com/olivia-marcus-survivor-hoboken/`.
- JSON-LD: `Person` for Olivia Marcus (name + `sameAs` her Instagram `https://www.instagram.com/oliviamarcus/`), optionally wrapped in an `Article`/`CollectionPage`. (Skip `VideoObject` — the videos are third-party Instagram embeds, so Google generally won't credit them to your page.)
- Open Graph + Twitter card with title, description, bare-domain URL, and a representative image.
- Add the page to `sitemap.xml`; do NOT add it to header, footer, or mobile nav.
- Lazy-load any images, set explicit dimensions, compress.
